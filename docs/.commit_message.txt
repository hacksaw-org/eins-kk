Prefix: # xxxx

why: 
how: 
other: 

# ====例====
# feat_: #1 ○○を実装 // #n: issue num, 続けてwhat相当の内容を明記
#
# why: ■■■だったため // なぜこの対応をしたか
# how: ☆☆☆を利用している // 新たに使い始めたライブラリやフレームワークの機能など
# // (下記は複数行の書き方の例)
# other: // その他記載しておきたい情報
#   xxxx
#   xxxx

# ==== Prefix ====
# feat_: 新しい機能、仕様変更
# fix__: バグの修正
# docs_: ドキュメントのみの変更
# style: 空白、フォーマット、セミコロン追加など
# refa_: 仕様に影響がないコード改善(リファクタ)、ファイル名変更、ディレクトリ構成の変更など
# perf_: パフォーマンス向上関連
# test_: テスト関連
# chore: (意味: 雑用)ビルド、補助ツール、ライブラリ関連、自動生成ファイルのコミット、その他
